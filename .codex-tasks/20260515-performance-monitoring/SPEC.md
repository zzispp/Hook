# 性能监控模块实现规格

## 目标

新增管理员可见的性能监控模块，菜单位于概览下仪表盘之后，展示实时与快照指标，并支持 `today`、`7d`、`30d`、`all` 时间范围。

## 关键约束

- `all` 只读取性能监控快照保留期内的时间桶，不查询请求明细表。
- API 最大返回 `720` 个序列点，后端按范围选择安全粒度。
- 新增 `performance_monitoring_retention_days` 系统设置，默认 `30`，非法非正数拒绝。
- 清理 worker 每天删除早于保留天数的性能监控快照，不影响请求记录。
- 快照 worker 异步运行，不阻塞 LLM proxy 主请求链路。
- 无 mock 成功路径；不支持的主机指标必须显式暴露状态。

## 交付边界

- 后端表结构、类型、storage、worker、API、菜单/API 权限、i18n seed。
- 前端类型、actions、页面、系统设置清理字段、导航值。
- 单元测试覆盖范围选择、点数上限、系统设置校验、清理删除。

# 用户使用记录

## Goal

为普通用户新增“使用记录”页面和受限 API，用户只能查看自己的请求使用指标，且响应中不暴露提供商、Provider Key、候选链路、实际成本、利润、payload 等上游商业信息。

## Scope

- 后端新增用户侧 `/api/request-records` 读接口。
- 用户侧接口按当前登录用户过滤记录。
- 用户侧响应 DTO 仅包含：时间、模型、API 格式、类型、Tokens、缓存命中率所需 token 字段、费用、首字、总耗时。
- 前端新增 `/dashboard/usage-records` 页面，菜单显示“使用记录”。
- 复用现有格式化/分页/轮询工具，避免复用 admin 详情抽屉。

## Out Of Scope

- 不开放普通用户查看详情抽屉。
- 不改管理员“请求记录”的语义和字段。
- 不加入兼容、模拟或前端-only 脱敏。

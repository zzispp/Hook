# 卡密类型到账余额类型调整

## 目标

将卡密类型从保存默认充值/赠款金额，调整为只保存到账余额类型；生成卡密时输入单个金额，并由类型决定写入充值余额或赠款余额。

## 边界

- 后端类型创建/更新 payload、存储模型、baseline schema、校验逻辑保持一致。
- 具体卡密仍保存 `recharge_amount` 与 `gift_amount`，用于兑换到账和流水。
- 前端卡密类型弹窗只选择余额类型；生成弹窗只有一个金额输入框，label 跟随类型。
- 管理端文案继续通过后端 i18n seed 提供。

## 验收

- Rust 编译检查通过或暴露真实错误。
- 前端 lint/build 通过或暴露真实错误。
- 搜索确认类型默认金额字段不再出现在卡密类型链路中。

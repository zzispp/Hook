# 请求记录内容与清理策略

## Goal

参考 `/Users/bubu/ZwjProjects/aether`，在 Hook 中实现请求记录的请求头、请求体、响应体持久化，并在设置中新增“清理策略” section，支持分别配置请求记录整体清理时间，以及请求记录内容字段的清理时间。

## Boundaries

- 请求记录内容清理后，记录本身保留，但请求头、请求体、响应体为空。
- 不引入静默 fallback、模拟成功、兼容性补丁或额外业务规则。
- 先验证 Hook 现状与 aether 现有实现，再按 Hook 既有架构落地。

## Validation

- 后端静态检查或相关测试。
- 前端 lint/build 或相关类型检查。
- 必要时用代码搜索确认设置项读写链路与清理链路。
